# Архитектурное решение по трейсингу

## Системы для трейсинга
Заказ проходит через следующие компоненты, они могут стать причиной зависания заказа и должны быть включены в трейсинг:
 - Shop API
 - CRM API
 - MES API

Данные для трейсинга:
- идентификатор операции
- идентификатор подзапроса
- номер заказа
- название сервиса
- API endpoint
- идентфикатор инстанса при переходе на горизонтальное масштабирование
- версия приложения
- время получения запроса
- длительность обработки запроса
- идентификатор файла заказа
- статус заказа

## Мотивация
- Увеличить степень удовлетворения клиентов, обнаружив и исправив места зависания заказа
- Уменьшить time to market и количество багов в системе с помощью упрощения поиска ошибок в ПО
- Собрать бизнес-инсайты о времени исполнения заказов в целом и распределении этого времени между различными системами и этапами
- Уменьшить стоимость исправления ошибок путём сокращения трудозатрат работников

## Предлагаемое решение
[jewerly_c4_model.drawio](./jewerly_c4_model.drawio)

1. Развернуть систему сбора данных FluentBit 
2. Развернуть систему хранения данных OpenSearch 
3. Развернуть систему трассировки Jaeger 
4. Сконфигурировать Jaeger использовать OpenSearch в качестве системы хранения 
5. Сконфигурировать пайплайны FluentBit для приёма и передачи OpenTelemetry в OpenSearch 
6. В определённые для трейсинга системы - Shop API, CRM API, MES API требуется добавить инструментацию OpenTelemetry и настроить передачу данных в FluentBit 
7. На API участвующие в процессе создания, рассчёта и исполнения заказов требуется добавить отправку данных с указанными выше данными

## Компромиссы
- Отсутствие разработчиков с компетенцией в Java делает доработку онлайн-магазина и CRM невозможной без найма
- Описанный трейсинг не принесёт пользы, если ошибка случилась на frontend-части интернет-магазина и заказ не успел попасть в API
- Описанного трейсинга не достаточно для полного определения ошибок, т.к. логи сервисов с возможными ошибками не отправляются в трейсинг

## Безопасность
- Для доступа к данным трейсинга сотрудники должны использовать личные учётные записи
- Доступ к данным трейсинга через Jaeger должен предоставляться только сотрудникам Поддержки, Разработки и Администраторам
- Доступ к данным трейсинга через OpenSearch должен присутствовать только у Администраторов системы
- В данных трейсинга не должна содержаться чувствительная информация вроде персональных или финансовых данных
